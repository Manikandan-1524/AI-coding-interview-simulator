### Step 1: Restart/Redeploy on Railway

1. Go to **Railway Dashboard** → Your Project → **Backend** service
2. Click **"Deploy"** button to rebuild with new code
3. Wait 5-10 minutes for rebuild and redeploy

1. In Railway project, check you have **PostgreSQL** service running (green status)

3. Wait 1-2 minutes for database to initialize

### Step 3: Verify Environment Variables in Railway

Click **Backend** service → **Variables** tab → Verify these are set:

```

✅ GEMINI_API_KEY = [your actual key]

✅ CORS_ALLOWED_ORIGINS = https://your-vercel-url.vercel.app,http://localhost:3000

AUTO-PROVIDED BY RAILWAY (don't add):
✅ DATABASE_URL = [auto-set by PostgreSQL service]
✅ DB_USERNAME = [auto-set by PostgreSQL service]
✅ DB_PASSWORD = [auto-set by PostgreSQL service]
```

### Step 4: Check the Database Connection

The production profile now:
- Uses PostgreSQL driver (org.postgresql.Driver)
- Reads DATABASE_URL environment variable
- Automatically creates/updates tables on startup
- Has proper connection pooling

### Step 5: Monitor Logs

1. Backend service → **Logs** tab
2. Watch for this message:

## 🔍 Troubleshooting

### Still getting "Failed to configure a DataSource" error?

**Solution:**
1. Verify PostgreSQL service status is **Running** (green)
2. Verify **DATABASE_URL** shows actual connection string in Variables tab
3. If not present, add Database manually:
   - Click **"+ New"** → **"Database"** → **"PostgreSQL"**
   - Railway auto-populates DATABASE_URL
4. Redeploy backend service
5. Check logs again

### Getting "Connection refused" errors?

**Solution:**
1. This means DATABASE_URL format is wrong
2. It should look like: `postgresql://user:password@host:5432/db_name`
3. Check that PostgreSQL service is Running
4. Wait 2-3 minutes after creating database for it to fully initialize

### Application starts but can't insert data?

**Solution:**
1. Check permissions on PostgreSQL database
2. Verify `spring.jpa.hibernate.ddl-auto=create-drop` in production profile
3. Check logs for Hibernate errors
4. Verify database tables are being created

## ✅ What Changed:

### Before (Error):
```properties
# Tried to use MySQL with no driver on Railway
spring.datasource.url=${DATABASE_URL:jdbc:mysql://localhost:3306/...}
spring.datasource.driver-class-name=${DB_DRIVER:com.mysql.cj.jdbc.Driver}
```

### After (Fixed):
```properties
# Uses PostgreSQL driver and explicit configuration
spring.datasource.url=${DATABASE_URL}
spring.datasource.driver-class-name=org.postgresql.Driver
spring.jpa.database=postgresql
```
